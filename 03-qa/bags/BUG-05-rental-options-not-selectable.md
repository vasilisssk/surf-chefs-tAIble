# BUG-05: Нельзя выбрать варианты проката во фрагменте бронирования

## Сводная таблица

| Параметр | Значение |
|---|---|
| **ID бага** | BUG-05 |
| **Где был найден** | `fragment_booking.xml` — фрагмент Подтверждения бронирования (`BookingConfirmationFragment`) |
| **Критичность** | Major |
| **Приоритет** | High |
| **Статус** | Исправлен |
| **Компонент** | UI — Booking |
| **Окружение** | Android Emulator |

## Описание

Выпадающий список выбора пакета проката (`AutoCompleteTextView` с id `actv_rental_package`) во фрагменте бронирования (`fragment_booking.xml`) не позволял пользователю выбрать вариант проката. При нажатии на поле не отображался список доступных опций, либо список появлялся, но выбор не фиксировался и значение не подставлялось в поле.

## Шаги для воспроизведения

1. Войти в приложение.
2. Перейти на вкладку «Расписание» или «Главная».
3. Выбрать класс и перейти к бронированию.
4. На экране подтверждения бронирования нажать на поле «Пакет проката».
5. Выпадающий список не открывается или выбор опции не применяется.

## Ожидаемый результат

При нажатии на поле отображается выпадающий список доступных пакетов проката (например, «Базовый», «Стандарт», «Премиум»). После выбора пакета его название отображается в поле, а цена пакета добавляется к итоговой сумме.

## Фактический результат

Выбор вариантов проката недоступен — список не раскрывается или выбранное значение не сохраняется.

## Причина

`AutoCompleteTextView` для выбора пакета проката не был привязан к адаптеру с данными опций проката, либо данные не были загружены из API. Также возможно, что `TextInputLayout` использовал стиль `ExposedDropdownMenu`, но адаптер не был установлен программно в `BookingConfirmationFragment`.

## Исправление

Загружен список опций проката из API и установлен `ArrayAdapter` для `actv_rental_package`. Добавлен обработчик выбора (`setOnItemClickListener`), который обновляет описание пакета и пересчитывает итоговую стоимость бронирования.
