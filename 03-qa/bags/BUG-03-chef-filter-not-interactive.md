# BUG-03: Поле фильтра по шефу в Расписании не интерактивно

## Сводная таблица

| Параметр | Значение |
|---|---|
| **ID бага** | BUG-03 |
| **Где был найден** | `fragment_schedule.xml` — фрагмент Расписания (`ScheduleFragment`) |
| **Критичность** | Major |
| **Приоритет** | High |
| **Статус** | Исправлен |
| **Компонент** | UI — Schedule |
| **Окружение** | Android Emulator |

## Описание

Поле ввода для фильтрации классов по шефу (`MaterialAutoCompleteTextView` с id `actv_chef_filter`) во фрагменте Расписания (`fragment_schedule.xml`) не реагировало на нажатия пользователя. При тапе по полю не открывался выпадающий список доступных шефов, и пользователь не мог выбрать шефа для фильтрации расписания.

## Шаги для воспроизведения

1. Войти в приложение.
2. Перейти на вкладку «Расписание» в нижнем меню.
3. Нажать на поле «Фильтр по шефу».
4. Поле не раскрывает выпадающий список, ввод/выбор недоступен.

## Ожидаемый результат

При нажатии на поле отображается выпадающий список (`DropdownMenu`) с именами доступных шефов. Пользователь может выбрать шефа, и список классов отфильтруется.

## Фактический результат

Поле не интерактивно — выпадающий список не появляется, выбор шефа невозможен.

## Причина

`MaterialAutoCompleteTextView` не был корректно привязан к адаптеру с данными шефов, либо адаптер не был установлен. Также возможно, что `inputType="none"` в сочетании с отсутствующим `showDropDown()` вызовом приводило к блокировке взаимодействия.

## Исправление

Настроен `ArrayAdapter` с данными шефов и привязан к `actv_chef_filter` в `ScheduleFragment`. Добавлен вызов `showDropDown()` при фокусе на поле для корректного отображения выпадающего списка.
