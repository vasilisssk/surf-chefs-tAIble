# BUG-04: Кнопка «Назад» не работает при переходе к Подтверждению бронирования и Деталям заказа

## Сводная таблица

| Параметр | Значение |
|---|---|
| **ID бага** | BUG-04 |
| **Где был найден** | `activity_main.xml`, `fragment_booking.xml`, `fragment_class_detail.xml` — навигация внутри `NavHostFragment` |
| **Критичность** | Critical |
| **Приоритет** | High |
| **Статус** | Исправлен |
| **Компонент** | Navigation — Main Activity |
| **Окружение** | Android Emulator |

## Описание

Системная кнопка «Назад» (Android Back Button) не выполняла навигацию назад при нахождении пользователя на фрагментах «Подтверждение бронирования» (`BookingConfirmationFragment`) и «Детали класса» (`ClassDetailFragment`). Вместо возврата к предыдущему экрану нажатие кнопки «Назад» либо игнорировалось, либо закрывало приложение.

## Шаги для воспроизведения

1. Войти в приложение.
2. Перейти на вкладку «Расписание» или «Главная».
3. Выбрать любой доступный класс и нажать на него — открывается `ClassDetailFragment`.
4. Нажать системную кнопку «Назад».
5. Кнопка не работает — экран не меняется.
6. Аналогично: оформить бронирование, перейти на `BookingConfirmationFragment`.
7. Нажать системную кнопку «Назад» — экран не меняется.

## Ожидаемый результат

При нажатии кнопки «Назад» выполняется `NavController.navigateUp()` или `popBackStack()`, и пользователь возвращается на предыдущий экран.

## Фактический результат

Кнопка «Назад» не реагирует на нажатия на указанных фрагментах, навигация назад не выполняется.

## Причина

В `MainActivity` не была переопределена обработка `onBackPressed()` / `OnBackPressedCallback` для корректной делегации навигации через `NavController`. Либо `NavHostFragment` не был установлен как `defaultNavHost`, либо callback не был зарегистрирован на фрагментах.

## Исправление

Добавлен `OnBackPressedCallback` в `MainActivity`, который делегирует обработку кнопки «Назад» в `NavController.navHostFragment`. Проверено, что `app:defaultNavHost="true"` установлен для `NavHostFragment` в `activity_main.xml`.
