# BUG-02: Отсутствует поле «Фамилия» во фрагменте Профиля

## Сводная таблица

| Параметр | Значение |
|---|---|
| **ID бага** | BUG-02 |
| **Где был найден** | `fragment_profile.xml` — фрагмент Профиля (`ProfileFragment`) |
| **Критичность** | Major |
| **Приоритет** | High |
| **Статус** | Исправлен |
| **Компонент** | UI — Profile |
| **Окружение** | Android Emulator |

## Описание

Во фрагменте Профиля (`fragment_profile.xml`) отсутствовало поле ввода «Фамилия» (`last_name`). Было представлено только поле «Имя» (`first_name`), в то время как модель данных пользователя предполагала наличие обоих полей. Это приводило к невозможности просмотра и редактирования фамилии пользователя через интерфейс профиля.

## Шаги для воспроизведения

1. Войти в приложение под существующим пользователем.
2. Перейти в нижнем меню на вкладку «Профиль».
3. Обратить внимание на список полей профиля.
4. Поле «Фамилия» отсутствует в списке.

## Ожидаемый результат

В профиле отображаются поля: Имя, Фамилия, Email, Телефон, Аллергии.

## Фактический результат

Поле «Фамилия» отсутствует, доступно только поле «Имя».

## Причина

В layout-файле `fragment_profile.xml` не был добавлен `TextInputLayout` для поля фамилии, несмотря на то, что бэкенд и модель данных поддерживали это поле.

## Исправление

Добавлен блок `TextInputLayout` с id `til_last_name` и `TextInputEditText` с id `et_last_name` в `fragment_profile.xml`. В `ProfileViewModel` и `ProfileFragment` добавлена обработка загрузки и сохранения значения фамилии.
