# Задача 1: Генерация требований к MVP (User Stories и Use Cases)

## Цель
Сгенерировать с помощью ИИ полный набор пользовательских историй (User Stories) и вариантов использования (Use Cases) для MVP мобильного приложения кулинарной студии «Шеф-стол».

## Симптом / Проблема
Заказчик описал бизнес-процесс в свободной форме (бриф). Необходимо формализовать потребности в структурированные артефакты: бизнес-требования (BR), функциональные требования (FR), пользовательские истории (US) и варианты использования (UC).

## Входные данные
- Бриф от заказчика: `01-analysis/brief/brief-cooking.md`
- Уточнения скоупа (только клиентское Android-приложение, API, без интерфейсов шефа/админа)

## Выполнение

### Шаг 1. Выявление вопросов к заказчику
- Сформирован список уточняющих вопросов по доменному слою, противоречиям и пробелам.
- Результат: `01-analysis/analysis/analysis-questions.md`

### Шаг 2. Ответы на вопросы (автоматически сгенерированные)
- Сгенерированы логичные и технически обоснованные ответы на каждый вопрос.
- Результат: `01-analysis/analysis/analysis-answers-suggestions.md`

### Шаг 3. Бизнес-требования (BR)
- Сформулированы 10 бизнес-требований (BR-001 … BR-010).
- Результат: `01-analysis/requirements/business-requirements.md`

### Шаг 4. Функциональные требования (FR)
- Сформулированы 11 функциональных требований (FR-001 … FR-011).
- Результат: `01-analysis/requirements/functional-requirements.md`

### Шаг 5. Нефункциональные требования
- Отфильтрованы нерелевантные (безопасность платежей, масштабируемость и т. д.).
- Результат: `01-analysis/requirements/non-functional-requirements-updated.md`

### Шаг 6. User Stories (US)
- Сформулированы 12 пользовательских историй (US-001 … US-012):
  - US-001: Просмотр доступных классов
  - US-002: Бронирование класса
  - US-003: Управление моими бронированиями
  - US-004: Получение уведомлений
  - US-005: Указание аллергий
  - US-006: Оценка шефа
  - US-007: Повторное бронирование
  - US-008: Просмотр информации о прокате
  - US-009: Регистрация в приложении
  - US-010: Просмотр отзывов о классах
  - US-011: Получение информации об отмене класса
  - US-012: Использование мобильного приложения
- Результат: `01-analysis/requirements/user-stories.md`

### Шаг 7. Use Cases (UC)
- Сформулированы 12 вариантов использования (UC-001 … UC-012):
  - UC-001: Просмотр расписания классов
  - UC-002: Бронирование класса
  - UC-003: Отмена бронирования
  - UC-004: Регистрация пользователя
  - UC-005: Аутентификация пользователя
  - UC-006: Просмотр личного кабинета
  - UC-007: Оценка шефа
  - UC-008: Указание аллергий
  - UC-009: Управление прокатом
  - UC-010: Получение уведомлений
  - UC-011: Просмотр отзывов о шефах
  - UC-012: Повторная запись на класс
- Результат: `01-analysis/requirements/use-cases.md`

## Результат
Все артефакты успешно сгенерированы и размещены в `01-analysis/requirements/`. Пользовательские истории и варианты использования покрывают полный скоуп MVP клиентского приложения.
