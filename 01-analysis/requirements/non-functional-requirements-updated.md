# Нефункциональные требования (обновленные)

## NFR-002: Совместимость с платформами
**ID:** NFR-002
**Формулировка:** Приложение должно работать на устройствах с Android 9+.
**Приоритет:** Высокий
**Метрики:** Поддержка устройств с разрешением экрана не менее 720x1280 пикселей.

## NFR-004: Надежность
**ID:** NFR-004
**Формулировка:** Система должна обеспечивать доступность не менее 99% в рабочее время.
**Приоритет:** Высокий
**Метрики:** Время безотказной работы системы не менее 99% в месяц.

## NFR-006: Удобство использования
**ID:** NFR-006
**Формулировка:** Приложение должно быть интуитивно понятным для пользователей старше 18 лет без специального обучения.
**Приоритет:** Средний
**Метрики:** Время выполнения основных задач (бронирование, просмотр расписания) не более 60 секунд для новых пользователей.

## NFR-007: Совместимость с API
**ID:** NFR-007
**Формулировка:** Приложение должно корректно взаимодействовать с существующим API студии.
**Приоритет:** Высокий
**Метрики:** Поддержка всех контрактов API, валидация данных, корректная обработка ошибок.

## NFR-008: Локализация
**ID:** NFR-008
**Формулировка:** Приложение должно поддерживать русский язык как основной.
**Приоритет:** Высокий
**Метрики:** Все интерфейсы, уведомления и сообщения на русском языке.

## NFR-010: Хранение данных
**ID:** NFR-010
**Формулировка:** Приложение должно поддерживать кэширование основных данных для работы в режиме ограниченного доступа к интернету.
**Приоритет:** Средний
**Метрики:** Возможность просмотра уже загруженного расписания без подключения к интернету.

## NFR-012: Уведомления
**ID:** NFR-012
**Формулировка:** Система уведомлений должна обеспечивать доставку сообщений с вероятностью не менее 95%.
**Приоритет:** Средний
**Метрики:** Push-уведомления и SMS доставляются в течение 5 минут после события.
