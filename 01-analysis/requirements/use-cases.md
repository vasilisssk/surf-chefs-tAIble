# Use Cases

## UC-001: Просмотр расписания классов
**ID:** UC-001
**Название:** Просмотр доступных кулинарных классов
**Участники:** Клиент, Система
**Приоритет:** Высокий
**Основной поток:**
1. Клиент открывает приложение
2. Система отображает расписание на ближайшие 7 дней
3. Клиент просматривает доступные классы
4. Система отображает детали класса по запросу клиента

**Альтернативный поток A1:** Если нет доступных классов
1. Система отображает сообщение "Пока нет доступных классов"
2. Клиент может установить напоминание о появлении новых классов

## UC-002: Бронирование класса
**ID:** UC-002
**Название:** Запись на кулинарный класс
**Участники:** Клиент, Система
**Приоритет:** Высокий
**Основной поток:**
1. Клиент выбирает интересующий класс
2. Клиент указывает, будет использовать свое оборудование или взять напрокат
3. Клиент подтверждает бронирование
4. Система проверяет наличие свободных мест
5. Система создает бронирование и отправляет подтверждение

**Альтернативный поток A1:** Класс полностью забронирован
1. Система уведомляет клиента о том, что мест нет
2. Клиент может выбрать другой класс или дату

## UC-003: Отмена бронирования
**ID:** UC-003
**Название:** Отмена записи на класс
**Участники:** Клиент, Система
**Приоритет:** Высокий
**Основной поток:**
1. Клиент просматривает свои бронирования
2. Клиент выбирает бронирование для отмены
3. Система проверяет время до начала класса
4. Если отмена возможна (за 24+ часа), система подтверждает отмену
5. Система освобождает место и обновляет статус бронирования

**Альтернативный поток A1:** Поздняя отмена
1. Система уведомляет клиента о штрафных последствиях
2. Клиент подтверждает отмену
3. Система начисляет "фол" и отменяет бронирование

## UC-004: Регистрация пользователя
**ID:** UC-004
**Название:** Регистрация нового клиента
**Участники:** Новый пользователь, Система
**Приоритет:** Высокий
**Основной поток:**
1. Пользователь выбирает опцию регистрации
2. Система запрашивает имя, email и телефон
3. Пользователь вводит данные
4. Система проверяет уникальность email и телефона
5. Система создает учетную запись

## UC-005: Аутентификация пользователя
**ID:** UC-005
**Название:** Вход в систему
**Участники:** Клиент, Система
**Приоритет:** Высокий
**Основной поток:**
1. Клиент вводит email или телефон и пароль
2. Система проверяет учетные данные
3. При успешной проверке система предоставляет доступ к аккаунту

## UC-006: Просмотр личного кабинета
**ID:** UC-006
**Название:** Просмотр истории и настроек
**Участники:** Клиент, Система
**Приоритет:** Средний
**Основной поток:**
1. Клиент переходит в личный кабинет
2. Система отображает историю бронирований
3. Клиент может обновить личную информацию
4. Клиент может настроить уведомления

## UC-007: Оценка шефа
**ID:** UC-007
**Название:** Выставление оценки после класса
**Участники:** Клиент, Система
**Приоритет:** Средний
**Основной поток:**
1. После завершения класса клиенту доступна форма оценки
2. Клиент выбирает оценку от 1 до 5 звёзд
3. Система сохраняет оценку и обновляет рейтинг шефа
4. Оценка становится частью профиля шефа

**Альтернативный поток A1:** Клиент уже оценил этого шефа
1. Система уведомляет, что оценка уже выставлена
2. Клиент не может изменить оценку

## UC-008: Указание аллергий
**ID:** UC-008
**Название:** Указание медицинских ограничений
**Участники:** Клиент, Система
**Приоритет:** Средний
**Основной поток:**
1. Клиент переходит к разделу указания аллергий
2. Клиент выбирает из списка распространенных аллергенов
3. При необходимости клиент указывает другие ограничения в свободной форме
4. Система сохраняет информацию и передает шефу за 24 часа до класса

## UC-009: Управление прокатом
**ID:** UC-009
**Название:** Выбор комплекта проката
**Участники:** Клиент, Система
**Приоритет:** Средний
**Основной поток:**
1. При бронировании клиент видит варианты комплектов проката
2. Клиент выбирает подходящий комплект (Basic, Standard, Premium)
3. Система добавляет стоимость проката к стоимости бронирования
4. При посещении клиент получает выбранный комплект

## UC-010: Получение уведомлений
**ID:** UC-010
**Название:** Получение напоминаний и уведомлений
**Участники:** Клиент, Система
**Приоритет:** Средний
**Основной поток:**
1. За 24 часа до начала класса система отправляет push-уведомление
2. За 2 часа до начала система отправляет второе напоминание
3. При отмене класса система немедленно отправляет уведомление
4. Клиент получает уведомления на мобильное устройство

**Альтернативный поток A1:** Клиент отказался от уведомлений
1. Система продолжает отправлять важные уведомления (отмена класса)
2. Другие уведомления не отправляются

## UC-011: Просмотр отзывов о шефах
**ID:** UC-011
**Название:** Ознакомление с рейтингами и отзывами
**Участники:** Клиент, Система
**Приоритет:** Средний
**Основной поток:**
1. Клиент просматривает информацию о шефе
2. Система отображает среднюю оценку и количество отзывов
3. Клиент может увидеть подробные оценки по категориям
4. При наличии текстовых отзывов система может их отображать

## UC-012: Повторная запись на класс
**ID:** UC-012
**Название:** Быстрая запись на новый класс
**Участники:** Постоянный клиент, Система
**Приоритет:** Низкий
**Основной поток:**
1. Постоянный клиент выбирает новый класс
2. Система автоматически заполняет предпочтения (прокат, аллергии)
3. Клиент подтверждает бронирование
4. Система учитывает статус постоянного клиента при начислении бонусов
