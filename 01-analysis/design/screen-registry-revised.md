# Реестр экранов приложения "Кулинарная студия" (обновленный)

## Общая архитектура

### Основная навигация
- Основное приложение реализуется с использованием Bottom Navigation Activity
- Аутентификация (регистрация/вход) реализуется на отдельных Activity
- После успешной аутентификации пользователь направляется на основную Activity с Bottom Navigation

## Структура экранов

### 1. Экран регистрации (Registration Activity)
**Тип:** Отдельная Activity  
**Связанные требования:** FR-001  
**Связанные Use Case:** UC-004  
**Описание:** Экран для регистрации нового пользователя  
**Компоненты:**
- Поля ввода: имя, email, телефон, **пароль**
- Кнопка регистрации
- Ссылка на экран входа

### 2. Экран входа (Login Activity)
**Тип:** Отдельная Activity  
**Связанные требования:** FR-001  
**Связанные Use Case:** UC-005  
**Описание:** Экран для входа зарегистрированных пользователей  
**Компоненты:**
- Поля ввода: **email**, **пароль**
- Кнопка входа
- Ссылка на восстановление пароля
- Ссылка на регистрацию

### 3. Основная Activity с Bottom Navigation
**Тип:** Bottom Navigation Activity  
**Описание:** Главная Activity приложения с навигацией в нижней части экрана

#### 3.1. Навигационные элементы:
- **Главная (Home)** - отображение ближайших бронирований и доступных классов
- **Расписание (Schedule)** - отображение доступных классов на 7 дней
- **Мои бронирования (My Bookings)** - список текущих и прошедших бронирований
- **Профиль (Profile)** - информация о пользователе и настройки

---

## Экраны внутри Bottom Navigation

### 4. Экран главной страницы (Home Fragment)
**Тип:** Fragment (внутри Bottom Navigation Activity)  
**Связанные требования:** FR-002, FR-003, FR-004  
**Связанные Use Case:** UC-001, UC-002  
**Описание:** Главный экран, отображающий ближайшие бронирования и доступные классы  
**Компоненты:**
- Блок "Ближайшие бронирования" (список)
- Блок "Рекомендуемые классы" (карусель)
- **Кнопка "Посмотреть все классы"** - открывает Schedule Fragment для просмотра полного расписания
- Статус авторизации пользователя

### 5. Экран расписания (Schedule Fragment)
**Тип:** Fragment (внутри Bottom Navigation Activity)  
**Связанные требования:** FR-002  
**Связанные Use Case:** UC-001  
**Описание:** Отображение доступных кулинарных классов на 7 дней  
**Компоненты:**
- Фильтры (по типу класса, шефу, цене)
- Календарь на 7 дней
- Список/сетка классов по дням
- Индикаторы доступных мест
- **Кнопка "Забронировать"** при нажатии на класс - открывает Class Detail Fragment (внутри Schedule Fragment как bottom sheet или переход на отдельный экран)

### 6. Экран деталей класса (Class Detail Fragment/Activity)
**Тип:** Fragment (внутри Schedule Fragment как bottom sheet или отдельная Activity)  
**Связанные требования:** FR-002, FR-005  
**Связанные Use Case:** UC-002  
**Описание:** Детальная информация о классе с возможностью бронирования  
**Компоненты:**
- Название и описание класса
- Информация о шефе
- Дата и время
- Доступные места
- Варианты проката
- **Кнопка "Забронировать"** - открывает Booking Fragment (внутри Schedule Fragment как bottom sheet)
- Отзывы о шефу

### 7. Экран бронирования (Booking Fragment)
**Тип:** Fragment (внутри Schedule Fragment как bottom sheet или отдельная Activity)  
**Связанные требования:** FR-003, FR-005  
**Связанные Use Case:** UC-002  
**Описание:** Экран подтверждения бронирования  
**Компоненты:**
- Информация о классе
- Выбор проката (Basic/Standard/Premium)
- **Указание аллергий** - подтягивается из профиля пользователя, с возможностью редактирования в момент бронирования
- Подтверждение бронирования
- Информация об оплате (на месте)

### 8. Экран "Мои бронирования" (My Bookings Fragment)
**Тип:** Fragment (внутри Bottom Navigation Activity)  
**Связанные требования:** FR-004  
**Связанные Use Case:** UC-003  
**Описание:** Список текущих и прошедших бронирований  
**Компоненты:**
- Фильтры (все, предстоящие, прошедшие)
- Список бронирований с датой, статусом и классом
- **При нажатии на бронирование** - открывается Booking Detail (внутри этого же фрагмента как bottom sheet)
- Кнопка отмены бронирования (доступна до 24 часов до начала)
- Индикатор статуса бронирования

### 9. Экран профиля (Profile Fragment)
**Тип:** Fragment (внутри Bottom Navigation Activity)  
**Связанные требования:** FR-001, FR-006, FR-010  
**Связанные Use Case:** UC-006  
**Описание:** Информация о пользователе и настройки  
**Компоненты:**
- Личная информация (имя, email, телефон)
- **Информация об аллергиях** - редактируется здесь
- Настройки уведомлений
- Статус постоянного клиента
- Кнопка выхода из аккаунта

### 10. Экран уведомлений (Notifications Fragment)
**Тип:** Fragment (внутри Bottom Navigation Activity)  
**Связанные требования:** FR-008  
**Связанные Use Case:** UC-010  
**Описание:** Список полученных уведомлений  
**Компоненты:**
- Список уведомлений с датой и типом
- Возможность очистки уведомлений
- Настройки уведомлений

## Уточненная навигация и потоки

### Поток бронирования:
1. Пользователь на Schedule Fragment видит класс
2. Нажимает "Забронировать" на классе
3. Открывается Class Detail Fragment (bottom sheet)
4. Нажимает "Подтвердить бронирование"
5. Открывается Booking Fragment (bottom sheet)
6. Подтверждает бронирование
7. Бронирование создается и появляется в My Bookings Fragment

### Поток оценки шефа:
1. После завершения класса (по времени или по уведомлению)
2. Пользователь заходит в My Bookings Fragment
3. Видит завершенное бронирование
4. Нажимает на него -> Booking Detail
5. Видит кнопку "Оценить шефа"
6. Нажимает - открывается форма оценки (внутри Booking Detail как bottom sheet)
7. Выставляет оценку (1-5 звезд)

### Поток управления аллергиями:
1. Пользователь заходит в Profile Fragment
2. Редактирует информацию об аллергиях
3. При бронировании эти данные подтягиваются автоматически
4. Возможна корректировка в момент бронирования

## Спецификации для дизайнера

### 1. Цветовая палитра
- Основной цвет: #E74C3C (красный, как у кулинарного огня)
- Акцентный цвет: #3498DB (синий, для интерактивных элементов)
- Фоновый цвет: #FFFFFF (белый)
- Цвет текста: #2C3E50 (темно-серый)

### 2. Типографика
- Заголовки: Roboto Bold, 24sp
- Подзаголовки: Roboto Medium, 18sp
- Основной текст: Roboto Regular, 14sp
- Вспомогательный текст: Roboto Light, 12sp

### 3. Элементы интерфейса
- Кнопки: закругленные углы (8dp), тень при нажатии
- Карточки: закругленные углы (12dp), тень
- Bottom Navigation: высота 80dp, иконки 24x24
- Bottom Sheets: для детализации и подтверждений

### 4. Адаптивность
- Поддержка экранов от 720x1280 пикселей
- Правильное масштабирование элементов
- Адаптация под разные плотности экранов

### 5. Состояния элементов
- Активное состояние: основной цвет
- Неактивное состояние: #BDC3C7 (светло-серый)
- Ошибка: #E74C3C (красный)
- Успешное действие: #27AE60 (зеленый)

## Сценарии использования экранов

### Сценарий 1: Просмотр и бронирование класса
1. Пользователь заходит в Schedule Fragment
2. Выбирает интересующий класс
3. Нажимает "Забронировать" -> открывается Class Detail Fragment
4. Подтверждает бронирование -> Booking Fragment
5. Бронирование создается -> обновление My Bookings Fragment

### Сценарий 2: Управление бронированиями
1. Пользователь находится в My Bookings Fragment
2. Выбирает бронирование
3. Просматривает детали или отменяет (если доступно)
4. Статус обновляется в списке

### Сценарий 3: Управление профилем
1. Пользователь заходит в Profile Fragment
2. Редактирует личную информацию
3. Обновляет информацию об аллергиях
4. Эти данные используются при бронировании

## Требования к UX
- Минимизация числа шагов для основных действий
- Четкая визуализация статусов бронирования
- Интуитивная навигация
- Быстрый доступ к основным функциям
- Индикация доступных мест
- Уведомления о важных изменениях

## Замечания по ER-модели
Возможно добавление сущности Notification для отслеживания отправленных уведомлений пользователям, особенно для напоминаний и уведомлений о возможности оценки шефа после посещения класса. Однако, основная логика уведомлений может быть реализована на уровне UI без сохранения в локальной базе данных.
