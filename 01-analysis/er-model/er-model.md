# ER-модель системы "Кулинарная студия"

## Описание сущностей и отношений

### 1. Client (Клиент)
**Тип доступа:** Read/Write
**Описание:** Пользователь приложения, который записывается на кулинарные классы
**Атрибуты:**
- client_id (PK) - уникальный идентификатор клиента
- first_name - имя клиента
- last_name - фамилия клиента
- email - электронная почта
- phone - номер телефона
- registration_date - дата регистрации
- total_classes_attended - количество посещенных классов
- allergies - информация об аллергиях
- is_permanent_client - статус постоянного клиента

**Операции:**
- Чтение: все данные при входе в систему и просмотр профиля
- Запись: регистрация, обновление профиля, указание аллергий

### 2. Chef (Шеф)
**Тип доступа:** Read-only
**Описание:** Кулинарный инструктор, ведущий классы
**Атрибуты:**
- chef_id (PK) - уникальный идентификатор шефа
- name - имя шефа
- specialization - специализация
- rating - средний рейтинг
- total_reviews - количество отзывов
- bio - краткая биография

**Операции:**
- Чтение: отображение информации о шефе, просмотры расписания, отзывы

### 3. CookingClass (Кулинарный класс)
**Тип доступа:** Read-only
**Описание:** Конкретное занятие по кулинарии
**Атрибуты:**
- class_id (PK) - уникальный идентификатор класса
- title - название класса
- description - описание программы
- date_time - дата и время проведения
- duration - продолжительность в часах
- max_participants - максимальное количество участников
- available_seats - доступные места
- chef_id (FK) - идентификатор ведущего шефа
- class_type - тип класса (для новичков, продвинутые, с духовкой и т.д.)
- price - стоимость участия

**Операции:**
- Чтение: отображение расписания, выбор класса для бронирования

### 4. Booking (Бронирование)
**Тип доступа:** Read/Write
**Описание:** Запись клиента на кулинарный класс
**Атрибуты:**
- booking_id (PK) - уникальный идентификатор бронирования
- client_id (FK) - идентификатор клиента
- class_id (FK) - идентификатор класса
- booking_date - дата создания бронирования
- status - статус (CONFIRMED, CANCELLED_BY_CLIENT, CANCELLED_BY_STUDIO, ATTENDED, NO_SHOW)
- rental_package - тип проката (Basic, Standard, Premium)
- cancellation_date - дата отмены (если была отмена)
- penalty_points - количество штрафных "фолов" за позднюю отмену

**Операции:**
- Чтение: просмотр бронирований клиента, история посещений
- Запись: создание бронирования, отмена бронирования

### 5. RentalEquipment (Прокатное оборудование)
**Тип доступа:** Read-only
**Описание:** Оборудование, предоставляемое напрокат
**Атрибуты:**
- rental_id (PK) - уникальный идентификатор типа проката
- package_name - название пакета (Basic, Standard, Premium)
- description - описание содержимого
- price - стоимость аренды
- available_count - доступное количество

**Операции:**
- Чтение: отображение вариантов проката при бронировании

### 6. Review (Отзыв)
**Тип доступа:** Write-once (только создание, без редактирования)
**Описание:** Отзыв клиента о проведенном классе и шефе
**Атрибуты:**
- review_id (PK) - уникальный идентификатор отзыва
- client_id (FK) - идентификатор клиента
- chef_id (FK) - идентификатор шефа
- class_id (FK) - идентификатор класса
- rating - оценка (1-5)
- review_date - дата создания отзыва

**Операции:**
- Чтение: отображение рейтингов и отзывов о шефах
- Запись: создание отзыва после посещения класса (только один отзыв на класс)

## Отношения между сущностями

### 1. Client --(создает)--> Booking
- Один клиент может создать множество бронирований
- Тип: One-to-Many

### 2. CookingClass --(имеет)--> Booking
- Один класс может иметь множество бронирований
- Тип: One-to-Many

### 3. Chef --(ведет)--> CookingClass
- Один шеф может вести множество классов
- Тип: One-to-Many

### 4. Client --(оставляет)--> Review
- Один клиент может оставить множество отзывов
- Тип: One-to-Many

### 5. Chef --(получает)--> Review
- Один шеф может получить множество отзывов
- Тип: One-to-Many

### 6. CookingClass --(требует)--> RentalEquipment
- Один класс может быть связан с несколькими пакетами проката
- Тип: Many-to-Many (через атрибут в Booking)

## Классификация сущностей по типу доступа

### Read-only сущности:
- Chef: информация о шефах поступает из существующего бэкенда, клиентское приложение только отображает
- CookingClass: расписание и информация о классах поступают из бэкенда, клиент не может их создавать или редактировать
- RentalEquipment: информация о доступном оборудовании для проката поступает из бэкенда

### Read/Write сущности:
- Client: клиент может регистрироваться, обновлять свой профиль
- Booking: клиент может создавать и отменять бронирования
- Review: клиент может создавать отзывы (один раз на посещенный класс)

### Write-once сущности:
- Review: после создания отзыв не может быть изменен или удален клиентом
