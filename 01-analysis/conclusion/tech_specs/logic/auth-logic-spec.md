## Обзор

Бизнес-логика аутентификации управляет процессами регистрации, входа и выхода пользователей в приложении "Кулинарная студия". Логика обеспечивает безопасное хранение токенов, проверку валидности сессии и обработку ошибок аутентификации.

### User Story

> Как пользователь, я хочу безопасно регистрироваться и входить в приложение,
> чтобы получить доступ к персонализированным функциям сервиса.

### Бизнес-ценность

- Обеспечение безопасности данных пользователей
- Поддержка персонализированного опыта
- Контроль доступа к функциям приложения

## Описание логики

### Регистрация (Register)

Процесс регистрации нового пользователя включает валидацию введенных данных, отправку запроса на сервер и сохранение полученного токена.

### Вход (Login)

Процесс аутентификации существующего пользователя включает проверку учетных данных, получение токена и его безопасное хранение.

### Выход (Logout)

Процесс деаутентификации включает удаление токена и очистку данных сессии.

## API запросы

### POST /auth/register

**Триггер:** Нажатие на кнопку регистрации

**Headers:**

| Поле | Описание |
|------|----------|
| `content-type` | application/json |

**Параметры/Body:**

| Параметр | Тип | Описание | Значение/Источник |
|----------|-----|----------|-------------------|
| `firstName` | string | Имя пользователя | Ввод пользователя |
| `email` | string | Email пользователя | Ввод пользователя |
| `phone` | string | Номер телефона | Ввод пользователя |
| `password` | string | Пароль пользователя | Ввод пользователя |

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| Загрузка | Индикатор загрузки на кнопке |
| Успех (201) | Сохранение токена и данных пользователя, переход к основному экрану |
| Ошибка 4xx | Отображение ошибки валидации |
| Ошибка 5xx | Снек "Произошла ошибка. Попробуйте позже" |
| Ошибка сети | Снек "Нет соединения. Проверьте подключение к интернету" |

## Критерии приёмки

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** пользователь вводит корректные данные для регистрации, **Когда** нажимает "Зарегистрироваться", **Тогда** создается новый аккаунт и происходит вход |
| AC-002 | **Дано** пользователь вводит корректные учетные данные, **Когда** нажимает "Войти", **Тогда** происходит успешная аутентификация |
| AC-003 | **Дано** пользователь авторизован, **Когда** нажимает "Выйти из аккаунта", **Тогда** происходит деаутентификация и очистка данных сессии |
| AC-N01 | **Дано** пользователь вводит некорректные данные, **Когда** пытается зарегистрироваться, **Тогда** отображается сообщение об ошибке |
| AC-N02 | **Дано** пользователь вводит неверные учетные данные, **Когда** пытается войти, **Тогда** отображается сообщение об ошибке аутентификации |

## Обработка ошибок

| Тип ошибки | Контекст | Действие |
|------------|----------|----------|
| Невалидные данные | Регистрация/вход | Отображение конкретных сообщений об ошибках валидации |
| Неверные учетные данные | Вход | Отображение общего сообщения "Неверный email или пароль" |
| Ошибка сети | Любой запрос | Отображение сообщения "Нет соединения. Проверьте подключение к интернету" |
| Серверная ошибка | Любой запрос | Отображение сообщения "Произошла ошибка. Попробуйте позже" |
