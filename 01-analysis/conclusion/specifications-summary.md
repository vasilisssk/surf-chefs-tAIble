# Сводка технических спецификаций приложения "Кулинарная студия"

## Обзор

Данный документ представляет собой сводку всех технических спецификаций для мобильного приложения "Кулинарная студия". Спецификации охватывают все аспекты приложения: от экранов пользовательского интерфейса до бизнес-логик.

## Архитектура приложения

### Общая структура
- Приложение построено по модульной архитектуре с четким разделением на экраны и бизнес-логики
- Используется Bottom Navigation для основной навигации между ключевыми функциями
- Аутентификация реализована через отдельные экраны регистрации и входа

### Навигационная структура
1. **Аутентификационные экраны** (отдельные Activity):
   - Экран регистрации (SCR-001)
   - Экран входа (SCR-002)

2. **Основная навигация** (Bottom Navigation Activity):
   - Главная (Home) - SCR-003
   - Расписание (Schedule) - SCR-004
   - Мои бронирования (My Bookings) - SCR-007
   - Профиль (Profile) - SCR-008
   - Уведомления (Notifications) - SCR-009

## Спецификации экранов

### Аутентификация
- **Экран регистрации** (SCR-001): Обработка регистрации новых пользователей с валидацией данных
- **Экран входа** (SCR-002): Аутентификация существующих пользователей

### Основной функционал
- **Экран главной страницы** (SCR-003): Отображение ближайших бронирований и рекомендуемых классов
- **Экран расписания** (SCR-004): Просмотр доступных классов с возможностью фильтрации
- **Экран деталей класса** (SCR-005): Подробная информация о классе с возможностью бронирования
- **Экран бронирования** (SCR-006): Подтверждение бронирования с выбором пакета проката и аллергиями
- **Экран моих бронирований** (SCR-007): Управление бронированиями с возможностью отмены и оценки

### Дополнительный функционал
- **Экран профиля** (SCR-008): Управление личной информацией и настройками
- **Экран уведомлений** (SCR-009): Просмотр истории уведомлений

## Бизнес-логики

### Основные логики
- **Аутентификация** (LOGIC-001): Регистрация, вход, выход пользователей
- **Бронирование** (LOGIC-002): Создание, отмена, управление бронированиями
- **Отзывы** (LOGIC-003): Создание и отображение отзывов о шефах

## Интеграции с API

Приложение интегрируется с REST API по следующим основным маршрутам:
- `/auth` - аутентификация пользователей (регистрация, вход, выход)
- `/profile` - управление профилем пользователя
- `/classes` - получение информации о классах
- `/bookings` - управление бронированиями
- `/chefs` - информация о поварах
- `/reviews` - система отзывов
- `/rental-packages` - аренда оборудования

- Заголовки: Roboto Bold, 24sp

## Потоки взаимодействия

### Поток бронирования:
1. Пользователь на Schedule Screen видит класс
2. Нажимает "Забронировать" на классе
3. Открывается Class Detail Screen
4. Нажимает "Подтвердить бронирование"
5. Открывается Booking Screen
6. Подтверждает бронирование
7. Бронирование создается и появляется в My Bookings Screen

### Поток оценки шефа:
1. После завершения класса (по времени)
2. Пользователь заходит в My Bookings Screen
3. Видит завершенное бронирование
4. Нажимает на него
5. Видит кнопку "Оценить шефа"
6. Нажимает - открывается форма оценки
7. Выставляет оценку (1-5 звезд)

## Технические особенности

### Безопасность
- Использование JWT токенов для аутентификации
- Безопасное хранение токенов в защищенном хранилище
- Проверка валидности токенов перед каждым запросом

### Производительность
- Кэширование часто используемых данных
- Ленивая загрузка контента
- Оптимизация сетевых запросов

### Обработка ошибок
- Комплексная обработка сетевых ошибок
- Пользовательские сообщения об ошибках
- Возможность повторных попыток после ошибок
