- **Цель:** Автоматизация процесса записи на кулинарные классы

## Противоречия и неясности в брифе

### 1. Уведомления и алгоритмы
В брифе говорится: "напоминать про запись" и "спрашивать про аллергии", но неясно, как это будет реализовано технически и когда именно.

### 2. Оплата
С одной стороны, говорится: "хотелось бы чтобы оплата была сразу", с другой - "пока принимаю наличкой и переводом". Неясно, какая модель оплаты будет на старте.

### 3. Отмена класса
Неясно, как будет происходить возмещение средств при отмене класса по инициативе студии.

### 4. Количество мест
В брифе упоминается: "рабочих столов всего 12", но также "на классы с духовкой и сложной техникой больше 8 человек не берём". Нужно уточнить, как это будет учитываться при бронировании.

## Доменный слой

### Основные сущности

#### 1. Клиент (Client)
- Идентификатор
- Личные данные (имя, контактная информация)
- История бронирований
- Предпочтения (аллергии, диетические ограничения)
- Статус постоянного клиента

#### 2. Кулинарный класс (CookingClass)
- Идентификатор
- Название и описание программы
- Дата и время проведения
- Максимальное количество участников
- Тип (новички, опытные, с духовкой и т.д.)
- Стоимость участия

#### 3. Шеф (Chef)
- Идентификатор
- Имя и специализация
- Расписание занятости
- Рейтинг и отзывы

#### 4. Рабочее место (Workstation)
- Идентификатор
- Тип (с духовой печью, с дополнительным оборудованием и т.д.)
- Статус доступности

#### 5. Бронирование (Booking)
- Идентификатор
- Клиент
- Класс
- Статус (подтверждено, отменено, отменено студией)
- Дата создания
- Дата отмены (если применимо)
- Информация о прокате

#### 6. Прокатное оборудование (RentalEquipment)
- Идентификатор
- Тип (ножи, фартуки, наборы)
- Статус (в наличии, в аренде)
- Стоимость аренды

#### 7. Отзыв (Review)
- Идентификатор
- Клиент
- Шеф
- Оценка
- Комментарий
- Дата

### Основные процессы

#### 1. Бронирование класса
1. Клиент просматривает доступные классы
2. Выбирает класс и дату
3. Указывает предпочтения (прокат, аллергии)
4. Подтверждает бронирование
5. Получает подтверждение и уведомление

#### 2. Отмена бронирования
1. Клиент выбирает бронирование для отмены
2. Проверяется возможность отмены (в зависимости от времени до начала)
3. Подтверждается отмена
4. Место становится доступным
5. При необходимости происходит возврат средств

#### 3. Оценка шефа
1. После завершения класса клиент может оценить шефа
2. Клиент указывает оценку и комментарий
3. Оценка сохраняется и влияет на рейтинг шефа

#### 4. Управление расписанием
1. Администратор создает/редактирует классы
2. Назначает шефов и рабочие места
3. Устанавливает максимальное количество участников
4. Публикует расписание

### Ограничения и правила

1. **Ограничение по времени отмены:** бронирования за менее чем X часов до начала нельзя отменить
2. **Ограничение по местам:** невозможно забронировать место, если класс полностью заполнен
3. **Ограничение на дублирование:** один клиент не может забронировать несколько мест на один класс (если не указано иное)
4. **Ограничение на прокат:** количество доступных наборов оборудования ограничено
5. **Ограничение на оценки:** можно оценить шефа только после посещения класса

### Возможные состояния сущностей

#### Статусы бронирования:
- Создано (CREATED)
- Подтверждено (CONFIRMED)
- Оплачено (PAID)
- Отменено клиентом (CANCELLED_BY_CLIENT)
- Отменено студией (CANCELLED_BY_STUDIO)
- Посещено (ATTENDED)
- Пропущено (NO_SHOW)

#### Статусы класса:
- Запланирован (SCHEDULED)
- Полностью забронирован (FULLY_BOOKED)
- Отменен (CANCELLED)
- Проведен (COMPLETED)
